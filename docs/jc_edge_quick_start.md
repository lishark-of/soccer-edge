# JC Edge 快速使用说明

JC Edge 是一个本地赛前研究工作台。它的重点不是让页面堆满数据，而是帮助使用者快速回答三个问题：

1. 今天有没有值得继续观察的比赛？
2. 当前信号为什么出现，可信度够不够？
3. 是否应该只看单关，还是可以进入组合观察？

## 打开 App 后先看哪里

默认先看「今日观察」。

这一页应该只解决最重要的事情：

- 今日或下一可售比赛有多少场
- 数据源是否可用
- 当前可信度等级
- Top 单关观察
- Top 2串1观察或拒绝原因
- Top 比分和总进球倾向
- 缺失情报会不会影响判断
- 严厉交易者结论

如果首页显示「暂不强行串联」，这不是 App 失败，而是纪律门控生效。它的意思是：当前信息不足，组合风险大于模型优势，宁可等待更多情报，也不要为了凑组合而降低质量。

## 建议的三步流程

1. 先点「运行赛前优化」

查看当前比赛中是否有正 EV、正 Edge、市场和模型方向一致的观察信号。

2. 再看「组合审核」

重点看为什么 2串1 或 3串1 被拒。优秀组合不是赔率越高越好，而是每一腿都要通过可信度、风险、相关性和信息完整度检查。

3. 最后做「赛后学习」

赛后把结果补进学习流程，才能逐步复盘 Brier、Log Loss、收益曲线和组合质量。没有赛后记录，模型只能停留在赛前判断，无法真正提高。

## 如何理解可信度

可信度不是胜率。

可信度表示当前信号背后的数据是否足够完整，主要看：

- 赔率数据是否真实可用
- 市场概率和模型概率是否一致
- 伤停、首发、天气、新闻面是否有可靠来源
- 历史回测样本是否足够
- 当前信号是否来自高赔率冷门
- 串联是否明显放大风险

如果可信度低，即使 EV 很高，也只能视为弱观察。

## 为什么高 EV 不一定值得组合

高 EV 可能来自模型和市场的差异，但它不等于高命中。

尤其是赔率很高的冷门方向，模型可能认为它被低估，但长期波动会非常大。JC Edge 会把这类信号标记为「冷门观察」或「高风险观察」，通常不会把它当作串联核心。

## 为什么没有 2串1 或 3串1

没有组合通常有四类原因：

- 可信度门控关闭
- 单腿风险太高
- 组合命中概率不足
- 缺失情报太多

这时页面应该明确显示「暂无优秀串联观察」和拒绝原因，而不是空白表格。

## 比分和总进球怎么看

比分和总进球更多是节奏参考。

比分 Top 5 来自 Poisson/xG 与 Dixon-Coles 修正，能帮助判断比赛可能偏开放还是偏谨慎。但比分本身波动很大，如果没有官方比分赔率、首发、伤停和天气支持，不应视为强信号。

总进球比具体比分更稳一些，但仍需要看赔率是否支持，以及模型概率是否明显高于市场概率。

## 情报覆盖怎么看

情报覆盖页不是为了炫数据，而是为了告诉你哪些信息是真的、哪些只是兜底估算。

状态含义：

- 已确认：接口或用户补充提供了明确数据
- 已检查但未返回：查过，但接口暂时没有返回，不代表确认没有
- 兜底估算：用城市、球队或国家信息近似，可信度较低
- 未接入：当前没有可靠数据
- 用户补充：来自本地 external signals 文件

复杂情报如果缺失，系统会降低可信度，不会编造。

## DS Pro 在这里做什么

DS Pro 适合做研究摘要、拒绝原因解释、赛后复盘文字和情报梳理。

它不应该直接替代赔率、赛果、伤停、首发或天气数据。AI 解释可以帮助你看懂为什么保留或拒绝某个信号，但不能把低可信度信号变成高可信度信号。

## 严厉交易者标准

JC Edge 应该优先做到：

- 有强信号时清楚展示
- 没有强组合时直接说暂不组合
- 高赔率冷门不包装成稳健方向
- 缺失情报不编造
- 表格尽量收起，首页只给结论和下一步
- 每次赛后都能记录和复盘

## 安全边界

本工具仅做本地概率分析、纸面模拟和赛后复盘。

它不提供真实交易入口，不处理支付，不连接账户，也不提供任何自动执行能力。

最终决策必须由使用者结合合法渠道、真实信息和自身风险承受能力独立判断。
